# Гибкий экспорт DXF по шаблону имени — новый макрос

- **Дата:** 2026-06-06
- **Тип:** новый макрос (отдельный файл, не трогает существующий макрос пластин)
- **Новый файл-исходник:** `src/TemplateDxfExport.bas`
- **Существующий макрос пластин** (`src/Macro11.bas`) остаётся без изменений.

---

## 1. Контекст и цель

Существующий макрос пластин жёстко именует файлы из двух размеров (`D1@Эскиз1`, `D2@Эскиз1`) и толщины. Это не подходит для других типов деталей (например, гнутых уголков), где в имени должны быть свои меняющиеся значения (полки `100x100`, длина и т.д.), а сами эти числа в SolidWorks лежат в отдельных столбцах таблицы параметров (потому что задают геометрию).

**Цель:** новый макрос, который именует DXF по **шаблону**, заданному для конкретной детали. Один макрос работает для любых деталей (уголок, пластина, что угодно); по каждому заказу оператор только копирует числа в столбцы таблицы параметров.

Экспорт развёртки тот же (`ExportToDWG2` с флагом листового металла) — он корректно разворачивает и гнутые детали.

## 2. Принцип: шаблон имени

Для каждой детали один раз задаётся **шаблон имени файла** с подстановками `{…}`. Пример для уголка:

```
уголок_{D1@Эскиз1}x{D2@Эскиз1}_{Длина}_Qty_{Количество}
```

Для каждой конфигурации макрос подставляет значения и добавляет `.dxf`:

→ `уголок_100x100_300_Qty_8.dxf`

## 3. Источники значений (правило `@`)

Для каждой подстановки `{X}`:

| Вид `{X}` | Трактовка | Как читается |
|---|---|---|
| Содержит `@`, напр. `{D1@Эскиз1}` | **размер** детали | `swModel.Parameter("D1@Эскиз1").SystemValue * 1000` → мм, формат как в макросе пластин (точка-разделитель, без хвостовых нулей) |
| Без `@`, напр. `{Длина}`, `{Количество}` | **свойство** конфигурации | `CustomPropertyManager.Get4` — сначала на уровне конфигурации, при пустом — на уровне файла |

Так для каждого столбца оператор сам решает: брать число **прямо из размера** (тогда дублировать ввод не нужно — те же числа, что задают полки, попадают в имя) или из **свойства**.

## 4. Где хранится шаблон

Шаблон хранится **в самой детали** как свойство **уровня файла** (вкладка «Настройка»/Custom), имя свойства — `Шаблон_имени`. Задаётся один раз на деталь. Читается через `swModel.Extension.CustomPropertyManager("")`.

Одна кнопка-макрос работает для всех типов деталей — поведение определяет шаблон внутри детали.

## 5. Логика работы (по шагам)

1. Проверки: открыт `ActiveDoc`; это `swDocPART`; деталь сохранена. Иначе — сообщение, выход.
2. Прочитать шаблон из свойства файла `Шаблон_имени`. Если пусто — сообщение «Добавь свойство файла `Шаблон_имени`…», выход.
3. Проверить шаблон на парность скобок `{` `}`. Если нарушена — сообщение, выход.
4. Диалог выбора папки для DXF (старт — папка детали). Отмена → выход.
5. Запомнить исходную активную конфигурацию.
6. Цикл по `GetConfigurationNames`:
   1. `ShowConfiguration2` + `EditRebuild3`.
   2. Если в конфигурации нет листового металла → пропуск + в отчёт.
   3. Собрать имя из шаблона (подстановка `{…}` по §3). Если хоть одна подстановка не разрешилась (нет размера; см. §7) → ошибка + в отчёт, конфигурацию пропустить.
   4. Очистить имя от недопустимых символов, добавить `.dxf`, обеспечить уникальность (защита от перезаписи).
   5. Экспорт развёртки `ExportToDWG2(... swExportToDWG_ExportSheetMetal ...)`.
   6. Учесть успех/ошибку.
7. Вернуть исходную конфигурацию.
8. Отчёт: «Выгружено N / Пропущено / Ошибок» + список проблем.

## 6. Алгоритм подстановки имени

```
имя = шаблон
повторять, пока в "имя" есть "{":
    взять подстроку между первой парой { }
    если "}" не найдена → ошибка шаблона (выход на шаге 3)
    значение = РазрешитьПодстановку(плейсхолдер)   ' см. §3
    если значение = «не найдено» → вернуть признак ошибки конфигурации
    заменить "{плейсхолдер}" на значение
имя = Очистить(имя) & ".dxf"
```

- «Размер не найден» (`Parameter` вернул `Nothing`) → ошибка конфигурации (пропуск + отчёт).
- Свойство отсутствует/пусто → подставляется как есть (пустая строка допустима; оператор отвечает за заполнение столбцов).

## 7. Обработка ошибок и краевые случаи

| Ситуация | Поведение |
|---|---|
| Нет документа / не деталь / не сохранён | Сообщение, выход |
| Нет свойства `Шаблон_имени` | Сообщение с подсказкой, выход |
| Непарные `{` `}` в шаблоне | Сообщение, выход |
| Отмена выбора папки | Сообщение «Отменено», выход |
| В конфигурации нет листового металла | Пропуск + отчёт |
| Размер из `{…@…}` не найден | Ошибка конфигурации (пропуск) + отчёт |
| `ExportToDWG2` вернул `False` | Ошибка + отчёт |
| Совпадение имени файла | Суффикс имени конфигурации, затем счётчик `_2`, `_3` |

## 8. Настройки (константы вверху модуля)

- `TEMPLATE_PROP = "Шаблон_имени"` — имя свойства файла с шаблоном.

(Имена размеров больше не зашиты в макрос — они приходят из шаблона детали.)

## 9. Переиспользуемые помощники

Из макроса пластин копируются без изменений (новый модуль самодостаточен):
`FormatNum`, `SanitizeName`, `UniqueFilePath`, `GetFolder`, и обход элементов для проверки листового металла. Новый код: чтение шаблона (свойство файла), разбор `{…}`, разрешение подстановки (размер/свойство).

## 10. Вне рамок (YAGNI)

- Прямое чтение Excel-заказа — оператор по-прежнему копирует значения в таблицу параметров вручную.
- Подпапки/раскладка по толщине, материал — не делаем.
- Не трогаем макрос пластин `src/Macro11.bas` — он остаётся отдельным.

## 11. Критерии готовности

- На детали с заданным свойством `Шаблон_имени` и несколькими конфигурациями макрос создаёт по одному DXF на конфигурацию, имя — результат подстановки шаблона.
- Размеры (`{…@…}`) берутся из геометрии и совпадают с реальными значениями конфигурации; свойства (`{…}`) — из свойств.
- Тот же макрос на другой детали с другим шаблоном даёт другие имена без правки кода.
- Совпадающие имена не перезаписываются; после прогона активная конфигурация восстановлена; показан отчёт.

## 12. Допущения

- Деталь — листовой металл (есть развёртка); экспортируется плоская заготовка.
- Шаблон хранится как свойство файла `Шаблон_имени`; значения для подстановки — размеры (по полному имени, с `@`) или свойства конфигурации/файла.
- Толщина в имени появляется только если её упомянуть в шаблоне (отдельно не добавляется).
- Доставка: новый файл `src/TemplateDxfExport.bas` добавляется в существующий репозиторий `solidworks-dxf-batch-export`, README дополняется описанием второго макроса.

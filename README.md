# Пакетный экспорт развёрток в DXF — макросы SolidWorks

Макросы для SolidWorks: открывают деталь из листового металла, проходят по **всем её конфигурациям** и выгружают развёртку каждой в отдельный DXF (для лазерной/плазменной резки).

В репозитории два макроса:
- **Макрос 1 (пластины)** — `src/Macro11.bas`: имя из геометрии (длина/ширина/толщина) и количества. Просто, для пластин.
- **Макрос 2 (универсальный)** — `src/TemplateDxfExport.bas`: имя по **шаблону** из свойств детали. Для любых деталей (уголки, профили и т.п.).

---

## Макрос 1 (пластины): имя из геометрии — `src/Macro11.bas`

## Требования

- SolidWorks с поддержкой листового металла.
- Деталь `.sldprt` из листового металла с несколькими **конфигурациями** (обычно через таблицу параметров).
- Русский интерфейс SolidWorks (имена размеров вида `D1@Эскиз1`). При другом языке — поменяй имена в разделе [Настройка](#настройка).

## Скачать

- Кнопка **`Code` → `Download ZIP`** на странице репозитория, **или**
- `git clone https://github.com/<owner>/<repo>.git`

Нужен один файл: **`src/Macro11.bas`**.

## Установка макроса в SolidWorks

1. В SolidWorks: **`Сервис → Макрос → Создать…`** (Tools → Macro → New), сохрани новый макрос (например `PlateDxfExport.swp`). Откроется редактор VBA.
2. Открой `src/Macro11.bas` в текстовом редакторе (VS Code / Блокнот), выдели **весь** текст и скопируй.
3. В редакторе VBA открой модуль (слева в дереве, напр. `Module1`), выдели всё (`Ctrl+A`) и вставь скопированный код.
   - ⚠️ Вставляй именно **копированием-вставкой**, а не через `File → Import` — так кириллица (имена размеров, сообщения) не испортится. Строки `Attribute` в файле нет — он готов к вставке.
4. **`Debug → Compile`** — ошибок быть не должно.

## Подготовка детали

Макрос берёт размеры **из геометрии**, количество — **из свойства конфигурации**:

| В имени файла | Источник в детали |
|---|---|
| 1-е число | размер `D1@Эскиз1` |
| 2-е число | размер `D2@Эскиз1` |
| 3-е число (толщина) | элемент листового металла (Sheet-Metal) — читается автоматически |
| `Qty_…` | свойство конфигурации `Quantity` |

В таблице параметров заведи столбцы размеров (`D1@Эскиз1`, `D2@Эскиз1`, размер толщины листового металла) и один столбец-свойство `$PRP@Quantity`. Каждая строка = одна конфигурация = одна пластина.

> Толщину **задаёшь ты** (как размер листового металла в таблице), а макрос **читает её сам** из модели — отдельное свойство для толщины заводить не нужно.

## Запуск

1. Открой и **сохрани** деталь.
2. Запусти макрос: **`Сервис → Макрос → Запустить…`** (или `F5` в редакторе VBA).
3. Выбери папку для сохранения DXF.
4. В конце появится отчёт: сколько **выгружено / пропущено / с ошибкой**. После работы деталь возвращается на исходную конфигурацию.

## Имя файла

```
Plate_{размер1}x{размер2}x{толщина}_Qty_{количество}.dxf
```

Пример: `Plate_52x300x1.5_Qty_8.dxf` (52 и 300 — размеры из `D1@Эскиз1` / `D2@Эскиз1`, 1.5 — толщина, 8 — количество).

- Разделитель дробной части — точка; лишние нули убираются (`2.0 → 2`).
- При совпадении имён к файлу добавляется имя конфигурации (затем `_2`, `_3`) — **перезаписи нет**.

## Настройка

В начале модуля `src/Macro11.bas`:

```vba
Private Const LENGTH_DIM As String = "D1@Эскиз1"  ' первый размер
Private Const WIDTH_DIM  As String = "D2@Эскиз1"  ' второй размер
Private Const QTY_PROP   As String = "Quantity"   ' свойство количества
Private Const FILE_PREFIX As String = "Plate_"    ' префикс имени файла
```

Если в твоей детали размеры называются иначе — поменяй `LENGTH_DIM` / `WIDTH_DIM`.
Узнать полное имя размера: дважды кликни по размеру на модели → в свойствах размера есть «Полное имя» (или включи показ имён размеров).

## Если что-то не так

| Сообщение / симптом | Причина |
|---|---|
| «Открой деталь!» | Нет активной детали |
| «Это не деталь!» | Открыт не `.sldprt` |
| «Сначала сохрани файл!» | Деталь не сохранена на диск |
| «ПРОПУСК (нет листового металла)» | В конфигурации нет элемента листового металла / развёртки |
| «ОШИБКА (нет размера …)» | Не найден `D1@Эскиз1` / `D2@Эскиз1` — проверь имена в [Настройке](#настройка) |
| Во всех DXF одинаковая толщина | Толщину нужно читать иначе — заведи issue |

## Макрос 2 (универсальный): имя по шаблону — `src/TemplateDxfExport.bas`

Для деталей любого типа (уголки, профили и т.п.), где имя нужно собирать из меняющихся значений. Имя DXF задаётся **шаблоном** в самой детали — один макрос на все типы деталей.

### Шаблон

В свойствах файла детали (`Файл → Свойства → Настройка`) заведи свойство **`Шаблон_имени`**, например:

```
уголок_{D1@Эскиз1}x{D2@Эскиз1}_{Длина}_Qty_{Количество}
```

Правило подстановки:
- `{…@…}` (есть `@`) → значение **размера** из геометрии (в мм). Те же числа, что задают деталь, попадают в имя — дублировать не нужно.
- `{…}` (без `@`) → значение **свойства** (сначала конфигурации, потом файла).

Результат: `уголок_100x100_300_Qty_8.dxf`.

### Установка и запуск

Как Макрос 1, но вставляешь `src/TemplateDxfExport.bas` в новый макрос. Перед запуском убедись, что у детали задано свойство `Шаблон_имени` — если его нет, макрос подскажет, что добавить.

### Если у всех DXF одинаковое имя

Шаблон записан синтаксисом SolidWorks (в кавычках `"…"`) — тогда SolidWorks вычисляет его один раз, на все конфигурации. Используй **фигурные скобки** `{…}`: их SolidWorks не трогает, подстановку по каждой конфигурации делает макрос. Проверка: в «Вычисленном значении» свойства `Шаблон_имени` должны остаться **скобки**, а не числа.

Свойства-столбцы в таблице параметров (напр. количество) задавай с префиксом `$PRP@` (в русском интерфейсе — `$СВОЙСТВО@`), иначе SolidWorks их не примет: `$PRP@Количество` / `$СВОЙСТВО@Количество`.
